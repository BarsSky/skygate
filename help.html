Как начать пользоваться

Skygate — это web-интерфейс для управления Tailscale-нодой headscale. Через него вы добавляете свои устройства в общий tailnet и управляете ими.

Шаг 1. Установить Tailscale

  • Android: Google Play → "Tailscale" → Установить
  • iOS: App Store → "Tailscale" → Установить
  • Windows: tailscale.com/download/windows
  • macOS: App Store → "Tailscale"
  • Linux: curl -fsSL https://tailscale.com/install.sh | sh

Шаг 2. Войти в свой аккаунт

При первом запуске Tailscale попросит войти. Используйте:

  • Сервер: tsnet.skynas.ru (наш headscale)
  • Логин: имя пользователя, которое вам выдал админ (например ivan)

Tailscale откроет браузер → введите логин/пароль → устройство появится в tailnet.

Шаг 3. Найти свои устройства

Перейдите на Мои устройства — увидите все свои ноды со статусами (online/offline, IP, теги).

Шаг 4. Добавить ещё устройств

В разделе Мои устройства нажмите "Добавить устройство":

  • Введите имя (например my-laptop, my-phone)
  • Нажмите Создать — появится preauth key
  • Скопируйте команду tailscale up --login-server=https://tsnet.skynas.ru --authkey=hskey-auth-XXXX
  • Вставьте её в терминал нового устройства

Готово — устройство появится в списке.

По умолчанию ваши устройства получают тег tag:private — они видны только вам.

Exit node (выход в интернет через другую страну)

Exit node — это сервер в tailnet, через который весь ваш трафик в интернет идёт. Полезно для:

  • 🌍 Обход геоблокировок — IP другой страны (YouTube/Netflix EU через Германию)
  • 🇷🇺 Обход блокировок в РФ — Telegram, YouTube работают
  • 🛡️ Приватность — провайдер не видит какие сайты вы посещаете
  • 📡 Безопасность в публичном WiFi

Какие exit node доступны

ИмяГдеКогда использовать
karolinaГермания (Hetzner)Обход РФ-блокировок (Telegram, YouTube), европейский Netflix
emiliaРоссияЛокальные РФ сервисы (Госуслуги, Сбер), низкий пинг к мобильному оператору

Как включить exit node

  1. Откройте Tailscale app на устройстве
  2. ☰ Меню → Use exit node → выберите (karolina / emilia / sharlotta)
  3. Готово — весь трафик идёт через выбранный сервер

Как проверить что работает

Откройте 2ip.ru — должен показать IP страны exit node (например Германия для karolina).

Как отключить

Tailscale app → ☰ → None (Use exit node) → трафик снова идёт напрямую.

Внимание: при включённом exit node скорость = скорость до exit node. Для karolina (Германия) из РФ это обычно 50-100 Mbit/s.

Что такое DERP relay

Когда два Tailscale клиента не могут соединиться напрямую (CGNAT, файрвол, NAT), трафик идёт через DERP relay — промежуточный сервер.

Что это значит для вас

  • 🟢 Direct — устройства соединились напрямую. Быстро, низкий пинг.
  • 🟡 Через DERP — трафик идёт через relay-сервер. Медленнее (+50-200ms), но работает всегда.

Где смотреть какой режим

На странице Мои устройства — в таблице виден пинг и способ соединения.

Частые проблемы

🔴 "Не могу войти в Tailscale" / "Invalid login server"

  1. Убедитесь что ввели tsnet.skynas.ru как login server
  2. Проверьте интернет-соединение
  3. Попробуйте открыть headscale в браузере
  4. Свяжитесь с админом — возможно ваш аккаунт заблокирован

🔴 "Exit node выбран, но нет интернета"

  1. Проверьте что exit node online (см. Мои устройства → раздел "Доступные exit nodes")
  2. Попробуйте другой exit node
  3. Перезапустите Tailscale на устройстве
  4. Если не помогло — сообщите админу

🔴 "MagicDNS не работает" / не резолвит имена

На Android 4G иногда ломается. Решение:

  1. Tailscale app → Settings → Tailscale DNS → OFF
  2. Используйте IP напрямую: 100.64.0.X (видно на странице устройств)

🔴 "Устройство показывает online, но не пингуется"

  • Проверьте что Tailscale запущен на том устройстве
  • Устройство может быть в спящем режиме (ноутбук закрыт) — Tailscale засыпает
  • На Android: откройте Tailscale app — соединение восстановится

🟡 "Медленно работает интернет через exit node"

  • Попробуйте ближайший exit node
  • Проверьте пинг — если > 200ms, возможно идёт через DERP (не напрямую)
  • Перезапустите Tailscale
Не нашли решение? Напишите админу: Headplane

Что такое tailnet

Tailnet — это mesh-сеть на базе Tailscale. Устройства в tailnet получают IP из диапазона 100.64.0.0/10 и видят друг друга по этим адресам независимо от того, где они находятся физически.

Поток данных

skybars
100.64.0.5 emilia
100.64.0.3 (exit) internet
8.8.8.8
Когда skybars использует emilia как exit node, весь его трафик в интернет идёт через VPS emilia.

Кто в нашем tailnet

См. Все устройства:

  • karolina — VPS в Германии (Hetzner), exit node
  • emilia — VPS в РФ, exit node
  • sharlotta — VPS в РФ, exit node
  • skyworker — рабочая Windows машина
  • skybars — Android (mobile), CGNAT

Управление

  • Все устройства — полный список, теги, удаление
  • Пользователи — создание, ACL, ресеты паролей
  • DERP relay — статус, нагрузка, лента
  • Audit log — кто что делал

Как работает DERP

DERP (Designated Encrypted Relay for Packets) — это relay-сервер, через который проходит трафик когда прямое соединение между двумя Tailscale клиентами невозможно (CGNAT, файрвол, NAT).

Архитектура

┌──────────┐    WireGuard-over-WS    ┌─────────┐    WebSocket    ┌──────────┐
│ skybars  │ ────TLS 1.3 + WSS────────▶│  DERP   │─────────────────▶│  emilia  │
│ 4G CGNAT │   TCP 443 (как HTTPS)    │  relay  │  TCP 443        │  VPS EU  │
└──────────┘                          └─────────┘                  └──────────┘
                                            │
                                  видит только:
                                  • IP клиентов
                                  • размер пакетов
                                  • НЕ видит содержимое
                                  (TLS 1.3 + WireGuard)

Когда включается DERP

  • 🟢 Direct — клиенты могут соединиться напрямую (LAN, пробили NAT). Лучшая скорость.
  • 🟡 DERP relay — direct невозможен. Tailscale автоматически выбирает ближайший DERP. Медленнее (200-400ms), но работает.

Почему DERP устойчив к блокировкам (YouTube, Telegram в РФ)

  • TLS 1.3 — DPI не видит что внутри (только "клиент ↔ какой-то HTTPS сервер")
  • TCP 443 — РКН не может заблокировать (сломает весь интернет)
  • WebSocket — выглядит как обычное HTTPS соединение
  • Ротация IP — Tailscale имеет ~30 DERP серверов по миру, IP меняются
  • Custom DERP — если все публичные заблокируют, используется derp.skynas.ru (наш белый IP, не заблокируют)

Слабые места

  • ⚠️ Throttling — РКН может замедлять HTTPS к IP Tailscale (с 50 Mbit/s до 1-2 Mbit/s)
  • ⚠️ Блокировка по IP — Tailscale IP могут попасть под блокировку (но наш derp.skynas.ru останется)
  • ⚠️ Latency — DERP добавляет +100-300ms к round-trip

Exit node

Exit node — это устройство в tailnet, через которое другие клиенты выходят в интернет. У нас 3 exit node: karolina, emilia, sharlotta.

Зачем это нужно

  • 🌍 Обход геоблокировок — выход в интернет с IP другой страны (karolina = Германия → YouTube/Netflix EU)
  • 🛡️ Приватность — провайдер видит только зашифрованный трафик до Tailscale, не знает какие сайты
  • 📡 Безопасность — публичный WiFi не видит ваш трафик
  • 🇷🇺 Обход блокировок в РФ — Telegram, YouTube через karolina (Германия) работают без блокировок

Как включить

  1. Открой Tailscale app на устройстве
  2. ☰ Меню → Use exit node → выбери (emilia/karolina/sharlotta)
  3. Готово — трафик идёт через выбранный exit node

Лучший выбор для разных задач

ЗадачаРекомендация
YouTube / Telegram (обход РФ блокировок)karolina Германия
Дома (WiFi)любой LAN direct < 5ms
Вне дома (4G)emilia низкий RTT к мобильному оператору
Локальные РФ сервисыemilia или sharlotta

Что нужно для работы (на стороне сервера)

iptables -t nat -A POSTROUTING -s 100.64.0.0/10 -o $EXT_IF -j MASQUERADE
iptables -A FORWARD -i tailscale0 -o $EXT_IF -j ACCEPT
iptables -A FORWARD -i $EXT_IF -o tailscale0 -m state --state RELATED,ESTABLISHED -j ACCEPT

# ACL в headscale (ОБЯЗАТЕЛЬНО для Tailscale 1.50+):
{"action": "accept", "src": ["*"], "dst": ["autogroup:internet:*"]}

ACL (Access Control List)

ACL управляет тем, кто к кому может подключаться в tailnet. Редактируется в Headplane или в файле /etc/headscale/acl.yaml.

Текущий ACL

См. Audit log или напрямую в headscale.

Частые задачи

🔧 Разрешить exit node

{
  "action": "accept",
  "src": ["*"],
  "dst": ["autogroup:internet:*"]
}

Без этого правила exit node не виден клиентам (Tailscale 1.50+ требует явно разрешить).

🔧 Изолировать пользователей

{
  "action": "accept",
  "src": ["skyadmin"],
  "dst": ["*:*"]
},
{
  "action": "accept",
  "src": ["ivan"],
  "dst": ["ivan:*", "tag:public:*"]
}

ivan видит только свои устройства + exit node.

Диагностика

На exit node

# Проверить что exit node виден клиентам
sudo journalctl -u tailscaled --since 5m | grep "suggested exit"
# Если "suggested exit node: ()\" — ACL блокирует

# Счётчики трафика через exit node
iptables -t nat -L POSTROUTING -n -v | grep "100.64.0.0/10"
iptables -L FORWARD -n -v | grep "tailscale0"

# Проверить direct vs relay
tailscale status
# active; direct 95.165.170.190:60822   ← хорошо
# active; relay "waw"                    ← через DERP, медленно

На клиенте (Android)

# Проверить какой DERP используется
adb shell tailscale netcheck 2>/dev/null || tailscale netcheck

# Принудительно переключить exit node
adb shell tailscale set --exit-node=karolina

Частые проблемы

🔴 "Relay server unavailable Helsinki"

Tailscale не может достучаться до публичного DERP в Хельсинки. Либо direct connection работает, либо нужен DERP в РФ.

  • Проверь: Tailscale app → Settings → Use exit node — должен быть выбран
  • На exit node: tailscale status показывает offers exit node?
  • Отключи-включи Tailscale на устройстве

🔴 Exit node выбран, но нет интернета

На exit node не хватает iptables правил. Должны быть:

MASQUERADE -s 100.64.0.0/10
FORWARD -i tailscale0 -o $EXT_IF -j ACCEPT

Проверь через iptables -t nat -L POSTROUTING и iptables -L FORWARD.

🔴 "DNS Unavailable" на Android

Tailscale DNS на Android 4G часто ломается. Решение:

  • Tailscale app → Settings → Tailscale DNS → OFF
  • Использовать системный DNS
