# v0.16.0+ — Per-user subnets

**Date**: 2026-07-16 (created), 2026-07-17 (decisions confirmed)
**Roadmap**: see AGENTS.md, the "Per-user subnets + cross-subnet
exit-node sharing (v0.16+ candidate, TBD with operator)" entry.

The 8 design decisions below were confirmed by the operator on
2026-07-17. v0.16.0 work starts with the schema + CIDR
allocator; the actual sidecar container management ships in
v0.16.1.

---

## Decisions (confirmed 2026-07-17)

### 1. Subnet router location → **(b) Skygate Docker sidecar per user**

Each user gets a tiny `tailscale/tailscale` container on the
skygate host that runs the subnet router 24/7. State persists
across skygate restarts (tailscale state dir is bind-mounted
into a per-user subdir of `./data/subnets/<user_id>/`).

**Trade-off vs. (a):** N×resource overhead (one container per
user). For 10 users this is ~10 MB of RAM and a few hundred MB
of disk per user — negligible on the operator's VM.

**Trade-off vs. (c):** no SPOF. Each user's subnet is alive
even if the shared router container dies.

### 2. CIDR strategy → **/24 per user (`10.0.<uid>.0/24`)**

- 256 users max (one /24 in the 10.0.0.0/16 range)
- 254 usable IPs per user (operator + 253 devices)
- Schema has a `subnet_bits INTEGER NOT NULL DEFAULT 24` column
  so we can move to /28 later without a schema migration
  (just an allocator change)

### 3. Bot → **yes, add /mysubnet**

Command shape (parallel to /myexitnodes / /mysettings):
- `<b>your subnet</b>`
- Field: `CIDR: 10.0.42.0/24`
- Field: `status: pending|active|disabled`
- Field: `router: emilia-subnet (offline)` or `router: not yet provisioned`
- Section "shared with me" — empty list in v0.16.0, populated
  in v0.17.1 when cross-user sharing ships
- Section "I shared with" — same

### 4. Multi-plane → **add `control_plane_url` column from v0.16.0**

`user_subnets.control_plane_url TEXT NOT NULL DEFAULT ''` — the
plane this subnet lives in. Empty string = global plane (same
as `portal_users.headscale_url = ''`). Each user's subnet lives
in their own plane's CIDR space (no cross-plane subnet sharing
without VPN-mesh).

This respects the v0.12.0 per-user headscale config and the
v0.13.0 per-plane ACL generation. Per-plane ACL is computed
in the per-plane context, so per-subnet rules need to be
generated in the per-plane context too (v0.16.1+ work).

### 5. Migration → **(A) opt-in per user**

Each existing user keeps their flat `100.64.0.0/10` IP until
they explicitly opt in via `/admin/users/{id}/subnet`. On
opt-in:
- skygate creates a subnet row in `user_subnets` (status=pending)
- skygate creates a sidecar container (v0.16.1)
- skygate issues a preauth key tagged with `tag:subnet-router`
- user registers a node with that key on their sidecar
- skygate tags the node `tag:subnet-router` and approves
  the personal CIDR

The flat device stays alive (and the user can still use it
on `--accept-routes`) until the user manually removes it.
This mirrors the v0.12.0 "no forced migration" principle.

### 6. Cross-user sharing → **IP-level only in v0.17.1, DNS in v0.19.0**

v0.17.1 ships:
- ACL rule: `{src: ["<user_B>@tsnet"], dst: ["<user_A_subnet>:*"]}`
- Audit log: `subnet_share_granted` / `subnet_share_revoked`
- Bot: `/share_subnet <username>`, `/revoke_subnet <username>`
- Admin: share table on `/admin/users/{id}/subnet`

v0.19.0 ships:
- headscale 0.23+ "service" record (DNS: `<service>.<user>.tailnet.skynas.ru`
  → `<user_subnet>:<port>`)

### 7. tagOwners update → **v0.17.0**

The `tag:subnet-router` entry is added to `tagOwners` in
`GenerateACL()` in v0.17.0 (when cross-subnet exit-node sharing
is implemented). For v0.16.0 the tag is created on the headscale
node but doesn't need to be in `tagOwners` (only the operator
tags the node, no rule references the tag yet).

### 8. Tests → **unit + fake-headscale first, live opt-in per-user**

For v0.16.0:
1. Unit tests for the CIDR allocator:
   - Idempotent: same user_id returns same CIDR
   - No overlap: two user_ids never get the same CIDR
   - Out-of-range: user_id > 255 returns an error
   - Cleanup: deleting a subnet frees the CIDR for reuse
2. Integration with fake headscale:
   - create user_subnets row → fake headscale: create
     preauth key, simulate node registration + route
     approval → verify state
3. Live opt-in is gated behind a "Try it" button on
   `/admin/users/{id}/subnet` so the operator controls
   per-user rollout.
