Telegram QR: wrap image in anchor, extract deep-link helper

The QR on /my/telegram was a bare <img>, so clicking it on a
desktop browser triggered Chrome's image-search action
('Search image with Google' / right-click menu) instead of
opening the deep link. On mobile the QR is meant to be
scanned with the camera app — that path is unaffected — but
a user who clicks the QR on desktop (e.g. to preview the
link without scanning) got a useless action.

What this changes:

  * QR image is now wrapped in an <a href=\"https://t.me/...\">.
    On desktop, clicking the QR opens the deep link in the
    browser (which on Android, with verified app links, hands
    off to Telegram). On mobile, the camera app scans the
    visible image — the anchor is invisible to the scanner.
  * The cursor on the anchor is 'pointer' and the title
    attribute reads 'Open in Telegram' (i18n key
    my_telegram.qr_link_title) so the user gets clear
    affordance feedback.
  * The deep-link URL format is extracted into a new
    TelegramDeepLink(username, token) helper in
    handlers_telegram_link.go. Both the QR handler and the
    template's <a> tag now build the URL through the same
    function — if the format ever changes (e.g. switch to
    tg://resolve for better Android deep-link reliability,
    add utm_source for analytics), there's exactly one place
    to change it AND one place to test it.
  * The template adds a 1px border around the QR image so
    the clickable area is visible to the user (the white
    background blends with the page otherwise).

Tests:

  * 5 new unit tests in handlers_telegram_link_test.go:
    - TestTelegramDeepLinkShape              — canonical case
    - TestTelegramDeepLinkEmptyUsername      — no half-URLs
    - TestTelegramDeepLinkEmptyToken         — same, for token
    - TestTelegramDeepLinkNoSpecialsInToken  — pin format
      to 'no encoding' (Telegram's bot API takes /start
      payload verbatim; encoding would break it)
    - TestTelegramDeepLinkDoesNotUseTgScheme — tripwire
      against an unreviewed format change
  * All 5 PASS.
  * Plus the existing 9 admin_telegram tests still PASS
    (handler logic unchanged, just the URL construction).
  * Full ./... test run: all 8 packages PASS.
