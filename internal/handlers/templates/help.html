Что такое tailnet

Tailnet — это ваша персональная mesh-сеть на базе Tailscale. Устройства в tailnet получают IP из диапазона 100.64.0.0/10 и видят друг друга по этим адресам независимо от того, где они находятся физически.

Поток данных

skybars
100.64.0.5 emilia
100.64.0.3 (exit) internet
8.8.8.8
Когда skybars использует emilia как exit node, весь его трафик в интернет идёт через VPS emilia в Германии (или какой у нас регион).

Кто в нашем tailnet

5 устройств (см. Все устройства):

  • karolina — VPS в Германии (Hetzner), exit node
  • emilia — VPS в РФ, exit node
  • sharlotta — VPS в РФ, exit node
  • skyworker — рабочая Windows машина
  • skybars — Android (mobile), CGNAT

Как работает DERP

DERP (Designated Encrypted Relay for Packets) — это relay-сервер, через который проходит трафик когда прямое соединение между двумя Tailscale клиентами невозможно (CGNAT, файрвол, NAT).

Архитектура

┌──────────┐    WireGuard-over-WS    ┌─────────┐    WebSocket    ┌──────────┐
│ skybars  │ ────TLS 1.3 + WSS────────▶│  DERP   │─────────────────▶│  emilia  │
│ 4G CGNAT │   TCP 443 (как HTTPS)    │  relay  │  TCP 443        │  VPS EU  │
└──────────┘                          └─────────┘                  └──────────┘
                                            │
                                  видит только:
                                  • IP клиентов
                                  • размер пакетов
                                  • НЕ видит содержимое
                                  (TLS 1.3 + WireGuard)

Когда включается DERP

  • 🟢 Direct — клиенты могут соединиться напрямую (LAN, пробили NAT). Лучшая скорость.
  • 🟡 DERP relay — direct невозможен. Tailscale автоматически выбирает ближайший DERP. Медленнее (200-400ms), но работает.

Почему DERP устойчив к блокировкам (YouTube, Telegram в РФ)

  • TLS 1.3 — DPI не видит что внутри (только "клиент ↔ какой-то HTTPS сервер")
  • TCP 443 — РКН не может заблокировать (сломает весь интернет)
  • WebSocket — выглядит как обычное HTTPS соединение
  • Ротация IP — Tailscale имеет ~30 DERP серверов по миру, IP меняются
  • Custom DERP — если все публичные заблокируют, используется derp.skynas.ru (наш белый IP, не заблокируют)

Слабые места

  • ⚠️ Throttling — РКН может замедлять HTTPS к IP Tailscale (с 50 Mbit/s до 1-2 Mbit/s)
  • ⚠️ Блокировка по IP — Tailscale IP могут попасть под блокировку (но наш derp.skynas.ru останется)
  • ⚠️ Latency — DERP добавляет +100-300ms к round-trip

Exit node

Exit node — это устройство в tailnet, через которое другие клиенты выходят в интернет. У нас 3 exit node: karolina, emilia, sharlotta.

Зачем это нужно

  • 🌍 Обход геоблокировок — выход в интернет с IP другой страны (karolina = Германия → YouTube/Netflix EU)
  • 🛡️ Приватность — провайдер видит только зашифрованный трафик до Tailscale, не знает какие сайты
  • 📡 Безопасность — публичный WiFi не видит ваш трафик
  • 🇷🇺 Обход блокировок в РФ — Telegram, YouTube через karolina (Германия) работают без блокировок

Как включить

  1. Открой Tailscale app на устройстве
  2. ☰ Меню → Use exit node → выбери (emilia/karolina/sharlotta)
  3. Готово — трафик идёт через выбранный exit node

Лучший выбор для разных задач

Дома (WiFi)любой LAN direct < 5ms
Вне дома (4G)emilia низкий RTT к мобильному оператору

Что нужно для работы (на стороне сервера)

iptables -t nat -A POSTROUTING -s 100.64.0.0/10 -o $EXT_IF -j MASQUERADE
iptables -A FORWARD -i tailscale0 -o $EXT_IF -j ACCEPT
iptables -A FORWARD -i $EXT_IF -o tailscale0 -m state --state RELATED,ESTABLISHED -j ACCEPT

# ACL в headscale (ОБЯЗАТЕЛЬНО для Tailscale 1.50+):
{"action": "accept", "src": ["*"], "dst": ["autogroup:internet:*"]}

Диагностика

На exit node

# Проверить что exit node виден клиентам
sudo journalctl -u tailscaled --since 5m | grep "suggested exit"
# Если "suggested exit node: ()" — ACL блокирует

# Счётчики трафика через exit node
iptables -t nat -L POSTROUTING -n -v | grep "100.64.0.0/10"
iptables -L FORWARD -n -v | grep "tailscale0"

# Проверить direct vs relay
tailscale status
# active; direct 95.165.170.190:60822   ← хорошо
# active; relay "waw"                    ← через DERP, медленно

На клиенте (Android)

# Проверить какой DERP используется
adb shell tailscale netcheck 2>/dev/null || tailscale netcheck

# Принудительно переключить exit node
adb shell tailscale set --exit-node=karolina

Частые проблемы

🔴 "Relay server unavailable Helsinki"

Tailscale не может достучаться до публичного DERP в Хельсинки. Либо direct connection работает, либо нужен DERP в РФ.

  • Проверь: Tailscale app → Settings → Use exit node — должен быть выбран
  • На exit node: tailscale status показывает offers exit node?
  • Отключи-включи Tailscale на устройстве

🔴 Exit node выбран, но нет интернета

На exit node не хватает iptables правил. Должны быть:

MASQUERADE -s 100.64.0.0/10
FORWARD -i tailscale0 -o $EXT_IF -j ACCEPT

Проверь через iptables -t nat -L POSTROUTING и iptables -L FORWARD.

🔴 "DNS Unavailable" на Android

Tailscale DNS на Android 4G часто ломается. Решение:

  • Tailscale app → Settings → Tailscale DNS → OFF
  • Использовать системный DNS
