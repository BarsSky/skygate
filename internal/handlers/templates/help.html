Getting started

Skygate is a web interface for managing your own Tailscale server, headscale. Through it you add your devices to a shared tailnet and manage them.

Step 1. Install Tailscale

  • Android: Google Play → "Tailscale" → Install
  • iOS: App Store → "Tailscale" → Install
  • Windows: tailscale.com/download/windows
  • macOS: App Store → "Tailscale"
  • Linux: curl -fsSL https://tailscale.com/install.sh | sh

Step 2. Sign in to your account

On the first launch Tailscale will ask you to sign in. Use:

  • Server: headscale.example.com
  • Login: the username your admin gave you (e.g. user1)

Tailscale opens a browser → enter login/password → the device appears in the tailnet.

Step 3. Find your devices

Go to My devices — you'll see all your nodes with statuses (online/offline, IP, tags).

Step 4. Add more devices

On the My devices page click "Add device":

  • Enter a name (e.g. my-laptop, my-phone)
  • Click Create — a preauth key appears
  • Copy the command tailscale up --login-server=https://headscale.example.com --authkey=hskey-auth-XXXX
  • Paste it into a terminal on the new device

Done — the device appears in the list.

By default your devices get the tag:private tag — visible only to you.

Exit node (routing internet through another node)

Exit node is a server in the tailnet through which all your internet traffic goes. Useful for:

  • 🌍 Routing — IP different from your own
  • 🇷🇺 Routing — services reachable through the exit node
  • 🛡️ Privacy — your ISP doesn't see which sites you visit
  • 📡 Public WiFi safety

Which exit nodes are available

See the list of available exit nodes on the Exit nodes page. The tailnet administrator manages the set.

How to enable an exit node

  1. Open Tailscale app on your device
  2. ☰ Menu → Use exit node → pick the node you need
  3. Done — all traffic goes through the chosen server

How to verify it works

Open any IP-check service — it should show the IP of the exit node's country.

How to disable

Tailscale app → ☰ → None (Use exit node) → traffic flows direct again.

Note: with an exit node enabled, your speed is capped by the link to the exit node.

What DERP relay is

When two Tailscale clients can't connect directly (CGNAT, firewall, NAT), traffic flows through a DERP relay — an intermediate server.

What this means for you

  • 🟢 Direct — devices connected directly. Fast, low ping.
  • 🟡 Through DERP — traffic goes via a relay. Slower (+50-200ms), but always works.

Our DERP relay

The tailnet admin can run a custom DERP server for minimum ping within the region. Address and status — on the DERP page.

Useful links

  • Tailscale docs: DERP servers
  • MagicDNS — addressing nodes by name

Common problems

🔴 "Can't sign in to Tailscale" / "Invalid login server"

  1. Check that you entered the correct login server (ask the admin)
  2. Verify your internet connection
  3. Try opening headscale in your browser
  4. Contact the admin — your account may be deactivated

🔴 "Exit node selected, but no internet"

  1. Check that the exit node is online (see My devices)
  2. Try a different exit node
  3. Restart Tailscale on the device
  4. If nothing helps — notify the admin

🔴 "MagicDNS doesn't work" / names don't resolve

On Android 4G it sometimes breaks. Fix:

  1. Tailscale app → Settings → Tailscale DNS → OFF
  2. Use the IP directly: 100.64.0.X (visible on the devices page)

🔴 "Device shows online, but doesn't ping"

  • Check that Tailscale is running on that device
  • Device may be in sleep mode — Tailscale sleeps with it
  • On Android: open the Tailscale app — the connection will resume

🟡 "Internet through exit node is slow"

  • Try the nearest exit node
  • Check the ping — if > 200ms, traffic likely goes through DERP
  • Restart Tailscale
Didn't find a solution? Message the tailnet admin.

What tailnet is

Tailnet is a mesh network built on Tailscale. Devices in the tailnet get IPs in the 100.64.0.0/10 range and see each other at those addresses regardless of where they physically are.

Data flow

my-phone
100.64.0.5 exit-vps
100.64.0.3 (exit) internet
8.8.8.8
When my-phone uses exit-vps as an exit node, all of its internet traffic goes through the exit-vps VPS.

Management

  • All devices — full list, tags, delete
  • Users — create, ACL, password resets
  • DERP relay — status, load
  • Audit log — who did what

How DERP works

DERP (Designated Encrypted Relay for Packets) is a relay server through which traffic flows when a direct connection between two Tailscale clients is impossible (CGNAT, firewall, NAT).

Architecture

┌──────────┐    WireGuard-over-WS    ┌─────────┐    WebSocket    ┌──────────┐
│ my-phone │ ────TLS 1.3 + WSS────────▶│  DERP   │─────────────────▶│ exit-vps │
│ 4G CGNAT │   TCP 443 (like HTTPS)   │  relay  │  TCP 443        │  VPS EU  │
└──────────┘                          └─────────┘                  └──────────┘
                                            │
                                  sees only:
                                  • client IPs
                                  • packet sizes
                                  • NOT contents
                                  (TLS 1.3 + WireGuard)

When DERP kicks in

  • 🟢 Direct — clients can connect directly. Best speed.
  • 🟡 DERP relay — direct not possible. Tailscale automatically picks the nearest DERP. Slower, but always works.

DERP protocol details

  • TLS 1.3 — DPI cannot see inside (only "client ↔ HTTPS server")
  • TCP 443 — standard HTTPS port, ubiquitous
  • WebSocket — looks like a normal HTTPS connection
  • IP rotation — Tailscale has many DERP servers worldwide
  • Custom DERP — you can run your own relay on a white IP

Weak points

  • ⚠️ Throttling — ISP may slow down HTTPS to Tailscale IPs
  • ⚠️ IP filtering — public DERP servers may be unreachable from some networks
  • ⚠️ Latency — DERP adds +100-300ms to round-trip

Your own DERP relay

Run on a server with a public IP. Status and metrics: DERP

Exit node

Exit node is a device in the tailnet through which other clients go to the internet.

Why it's needed

  • 🌍 Routing — go to the internet with the exit-node's IP
  • 🛡️ Privacy — ISP only sees encrypted traffic to Tailscale
  • 📡 Security — public WiFi can't see your traffic
  • 🇷🇺 Routing — services reachable through the exit node

How to enable

  1. Open the Tailscale app on your device
  2. ☰ Menu → Use exit node → pick a node
  3. Done — traffic flows through the chosen exit node

What's needed on the server

iptables -t nat -A POSTROUTING -s 100.64.0.0/10 -o $EXT_IF -j MASQUERADE
iptables -A FORWARD -i tailscale0 -o $EXT_IF -j ACCEPT
iptables -A FORWARD -i $EXT_IF -o tailscale0 -m state --state RELATED,ESTABLISHED -j ACCEPT

ACL (Access Control List)

ACL controls who can connect to whom in the tailnet. Edit in Skygate or directly in headscale.

Allow exit node

{
  "action": "accept",
  "src": ["*"],
  "dst": ["autogroup:internet:*"]
}

Without this rule the exit node is invisible to clients (Tailscale 1.50+ requires explicit allow).

Isolate users

{
  "action": "accept",
  "src": ["admin"],
  "dst": ["*:*"]
},
{
  "action": "accept",
  "src": ["user1"],
  "dst": ["user1:*", "tag:public:*"]
}

user1 sees only their own devices + exit node.

Diagnostics

On an exit node

# Check that the exit node is visible to clients
sudo journalctl -u tailscaled --since 5m | grep "suggested exit"
# If "suggested exit node: ()" — ACL is blocking

# Traffic counters through exit node
iptables -t nat -L POSTROUTING -n -v | grep "100.64.0.0/10"
iptables -L FORWARD -n -v | grep "tailscale0"

# Direct vs relay
tailscale status
# active; direct 203.0.113.1:41641   ← good
# active; relay "fra"                  ← through DERP

Common problems

🔴 Exit node selected, no internet

The exit node is missing iptables rules. Check iptables -t nat -L POSTROUTING and iptables -L FORWARD.

🔴 "DNS Unavailable" on Android

Tailscale DNS on Android 4G frequently breaks. Fix: Tailscale app → Settings → Tailscale DNS → OFF.
