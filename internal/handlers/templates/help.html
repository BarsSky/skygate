Как начать пользоваться

Skygate — это web-интерфейс для управления своим Tailscale-сервером headscale. Через него вы добавляете свои устройства в общий tailnet и управляете ими.

Шаг 1. Установить Tailscale

  • Android: Google Play → "Tailscale" → Установить
  • iOS: App Store → "Tailscale" → Установить
  • Windows: tailscale.com/download/windows
  • macOS: App Store → "Tailscale"
  • Linux: curl -fsSL https://tailscale.com/install.sh | sh

Шаг 2. Войти в свой аккаунт

При первом запуске Tailscale попросит войти. Используйте:

  • Сервер: headscale.example.com
  • Логин: имя пользователя, которое вам выдал админ (например user1)

Tailscale откроет браузер → введите логин/пароль → устройство появится в tailnet.

Шаг 3. Найти свои устройства

Перейдите на Мои устройства — увидите все свои ноды со статусами (online/offline, IP, теги).

Шаг 4. Добавить ещё устройств

В разделе Мои устройства нажмите "Добавить устройство":

  • Введите имя (например my-laptop, my-phone)
  • Нажмите Создать — появится preauth key
  • Скопируйте команду tailscale up --login-server=https://headscale.example.com --authkey=hskey-auth-XXXX
  • Вставьте её в терминал нового устройства

Готово — устройство появится в списке.

По умолчанию ваши устройства получают тег tag:private — они видны только вам.

Exit node (выход в интернет через другой узел)

Exit node — это сервер в tailnet, через который весь ваш трафик в интернет идёт. Полезно для:

  • 🌍 Маршрутизация — IP, отличный от вашего
  • 🇷🇺 Маршрутизация — сервисы доступны через exit node
  • 🛡️ Приватность — провайдер не видит какие сайты вы посещаете
  • 📡 Безопасность в публичном WiFi

Какие exit node доступны

Список доступных exit-узлов смотрите на странице Exit nodes. Администратор tailnet настраивает их состав.

Как включить exit node

  1. Откройте Tailscale app на устройстве
  2. ☰ Меню → Use exit node → выберите нужный узел
  3. Готово — весь трафик идёт через выбранный сервер

Как проверить что работает

Откройте любой сервис проверки IP — должен показать IP страны exit-узла.

Как отключить

Tailscale app → ☰ → None (Use exit node) → трафик снова идёт напрямую.

Внимание: при включённом exit node скорость ограничена скоростью канала до exit-узла.

Что такое DERP relay

Когда два Tailscale клиента не могут соединиться напрямую (CGNAT, файрвол, NAT), трафик идёт через DERP relay — промежуточный сервер.

Что это значит для вас

  • 🟢 Direct — устройства соединились напрямую. Быстро, низкий пинг.
  • 🟡 Через DERP — трафик идёт через relay-сервер. Медленнее (+50-200ms), но работает всегда.

Наш DERP relay

Администратор tailnet может запустить собственный DERP-сервер для минимального пинга внутри региона. Адрес и статус — на странице DERP.

Полезные ссылки

  • Tailscale docs: DERP servers
  • MagicDNS — как обращаться к нодам по имени

Частые проблемы

🔴 "Не могу войти в Tailscale" / "Invalid login server"

  1. Убедитесь что ввели правильный login server (уточните у администратора)
  2. Проверьте интернет-соединение
  3. Попробуйте открыть headscale в браузере
  4. Свяжитесь с админом — возможно ваш аккаунт деактивирован

🔴 "Exit node выбран, но нет интернета"

  1. Проверьте что exit node online (см. Мои устройства)
  2. Попробуйте другой exit node
  3. Перезапустите Tailscale на устройстве
  4. Если не помогло — сообщите админу

🔴 "MagicDNS не работает" / не резолвит имена

На Android 4G иногда ломается. Решение:

  1. Tailscale app → Settings → Tailscale DNS → OFF
  2. Используйте IP напрямую: 100.64.0.X (видно на странице устройств)

🔴 "Устройство показывает online, но не пингуется"

  • Проверьте что Tailscale запущен на том устройстве
  • Устройство может быть в спящем режиме — Tailscale засыпает
  • На Android: откройте Tailscale app — соединение восстановится

🟡 "Медленно работает интернет через exit node"

  • Попробуйте ближайший exit node
  • Проверьте пинг — если > 200ms, возможно идёт через DERP
  • Перезапустите Tailscale
Не нашли решение? Напишите администратору tailnet.

Что такое tailnet

Tailnet — это mesh-сеть на базе Tailscale. Устройства в tailnet получают IP из диапазона 100.64.0.0/10 и видят друг друга по этим адресам независимо от того, где они находятся физически.

Поток данных

my-phone
100.64.0.5 exit-vps
100.64.0.3 (exit) internet
8.8.8.8
Когда my-phone использует exit-vps как exit node, весь его трафик в интернет идёт через VPS exit-vps.

Управление

  • Все устройства — полный список, теги, удаление
  • Пользователи — создание, ACL, ресеты паролей
  • DERP relay — статус, нагрузка
  • Audit log — кто что делал

Как работает DERP

DERP (Designated Encrypted Relay for Packets) — это relay-сервер, через который проходит трафик когда прямое соединение между двумя Tailscale клиентами невозможно (CGNAT, файрвол, NAT).

Архитектура

┌──────────┐    WireGuard-over-WS    ┌─────────┐    WebSocket    ┌──────────┐
│ my-phone │ ────TLS 1.3 + WSS────────▶│  DERP   │─────────────────▶│ exit-vps │
│ 4G CGNAT │   TCP 443 (как HTTPS)    │  relay  │  TCP 443        │  VPS EU  │
└──────────┘                          └─────────┘                  └──────────┘
                                            │
                                  видит только:
                                  • IP клиентов
                                  • размер пакетов
                                  • НЕ видит содержимое
                                  (TLS 1.3 + WireGuard)

Когда включается DERP

  • 🟢 Direct — клиенты могут соединиться напрямую. Лучшая скорость.
  • 🟡 DERP relay — direct невозможен. Tailscale автоматически выбирает ближайший DERP. Медленнее, но работает всегда.

Технические особенности протокола DERP

  • TLS 1.3 — DPI не видит что внутри (только "клиент ↔ HTTPS сервер")
  • TCP 443 — стандартный HTTPS-порт, используемый повсеместно
  • WebSocket — выглядит как обычное HTTPS соединение
  • Ротация IP — Tailscale имеет множество DERP серверов по миру
  • Custom DERP — можно запустить свой relay с белым IP

Слабые места

  • ⚠️ Throttling — провайдер может замедлять HTTPS к IP Tailscale
  • ⚠️ IP-фильтрация — публичные DERP-серверы могут быть недоступны из некоторых сетей
  • ⚠️ Latency — DERP добавляет +100-300ms к round-trip

Exit node

Exit node — это устройство в tailnet, через которое другие клиенты выходят в интернет.

Зачем это нужно

  • 🌍 Маршрутизация — выход в интернет с IP exit-узла
  • 🛡️ Приватность — провайдер видит только зашифрованный трафик до Tailscale
  • 📡 Безопасность — публичный WiFi не видит ваш трафик
  • 🇷🇺 Маршрутизация — сервисы доступны через exit node

Как включить

  1. Откройте Tailscale app на устройстве
  2. ☰ Меню → Use exit node → выберите узел
  3. Готово — трафик идёт через выбранный exit node

Что нужно для работы (на стороне сервера)

iptables -t nat -A POSTROUTING -s 100.64.0.0/10 -o $EXT_IF -j MASQUERADE
iptables -A FORWARD -i tailscale0 -o $EXT_IF -j ACCEPT
iptables -A FORWARD -i $EXT_IF -o tailscale0 -m state --state RELATED,ESTABLISHED -j ACCEPT

ACL (Access Control List)

ACL управляет тем, кто к кому может подключаться в tailnet. Редактируется в Skygate или напрямую в headscale.

Разрешить exit node

{
  "action": "accept",
  "src": ["*"],
  "dst": ["autogroup:internet:*"]
}

Без этого правила exit node не виден клиентам (Tailscale 1.50+ требует явно разрешить).

Изолировать пользователей

{
  "action": "accept",
  "src": ["admin"],
  "dst": ["*:*"]
},
{
  "action": "accept",
  "src": ["user1"],
  "dst": ["user1:*", "tag:public:*"]
}

user1 видит только свои устройства + exit node.

Диагностика

На exit node

# Проверить что exit node виден клиентам
sudo journalctl -u tailscaled --since 5m | grep "suggested exit"
# Если "suggested exit node: ()" — ACL блокирует

# Счётчики трафика через exit node
iptables -t nat -L POSTROUTING -n -v | grep "100.64.0.0/10"
iptables -L FORWARD -n -v | grep "tailscale0"

# Проверить direct vs relay
tailscale status
# active; direct 203.0.113.1:41641   ← хорошо
# active; relay "fra"                  ← через DERP

Частые проблемы

🔴 Exit node выбран, но нет интернета

На exit node не хватает iptables правил. Проверьте iptables -t nat -L POSTROUTING и iptables -L FORWARD.

🔴 "DNS Unavailable" на Android

Tailscale DNS на Android 4G часто ломается. Решение: Tailscale app → Settings → Tailscale DNS → OFF.
