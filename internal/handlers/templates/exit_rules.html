Как это работает

  1. Exit-узлы анонсируют все whitelist-подсети через --advertise-routes. Headscale раздаёт их клиентам по ACL.
  2. Skygate автоматически синхронизирует advertised-routes при добавлении/удалении правил.
  3. Клиенты:
    • Windows/Linux: tailscale up --accept-routes — split-туннель (только whitelist → exit-node)
    • Android/iOS: tailscale up --exit-node=<имя> — полный туннель

Быстрый старт

  1. Выберите устройство и exit-node
  2. Укажите тип и значение цели
  3. Нажмите «Добавить»
  4. На устройстве: tailscale up --accept-routes

Подключение клиента

Windows / Linux:

tailscale up --accept-routes

Без --exit-node — только whitelist-трафик идёт через туннель, остальное напрямую.

Android / iOS:

tailscale up --exit-node=exit-node-name

Полный туннель — избирательная маршрутизация не поддерживается на мобильных.

Важно

  • IP/подсеть — работает для split-туннеля (Windows/Linux).
  • Домен — только ACL-правило, не влияет на маршрутизацию.
  • После перезапуска Tailscale (tailscale down/up) выполните --accept-routes заново.
  • Правила изолированы: каждый пользователь управляет только своими устройствами.
