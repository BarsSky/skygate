1. Linux host setup as exit-node

    
Run on the exit-node (one-time):

    
# Enable IP forwarding
echo 'net.ipv4.ip_forward = 1' | sudo tee /etc/sysctl.d/99-tailscale-exit.conf
sudo sysctl -p /etc/sysctl.d/99-tailscale-exit.conf

# NAT for tailnet (100.64.0.0/10)
EXT_IF=$(ip route | awk '/default/ {print $5; exit}')
sudo iptables -t nat -A POSTROUTING -s 100.64.0.0/10 -o "$EXT_IF" -j MASQUERADE
sudo iptables -A FORWARD -i tailscale0 -o "$EXT_IF" -j ACCEPT
sudo iptables -A FORWARD -i "$EXT_IF" -o tailscale0 -m state --state RELATED,ESTABLISHED -j ACCEPT

# Persist rules
sudo mkdir -p /etc/iptables && sudo iptables-save | sudo tee /etc/iptables/rules.v4

  


  

    
2. Tailscale up with exit-node flags

    
Run on the exit-node after registering with headscale:

    
tailscale up --login-server={{.ControlURL}} --ssh --advertise-exit-node --accept-routes

    

      --ssh — Tailscale SSH for remote management

      --advertise-exit-node — act as exit node

      --accept-routes — accept routes from other nodes
    

  


  

    
3. Windows client: split-tunnel

    
On Windows, do NOT use --exit-node. Instead:

    
tailscale up --accept-routes

    

      --accept-routes accepts advertised routes from exit-nodes — only whitelisted IPs go through the tunnel.

      Important: do NOT use --exit-node on Windows — WireGuardNT (WFP) intercepts all traffic before the routing table, static routes don't work.
    

  


  

    
4. SSH access for auto-sync

    
Skygate uses SSH to update --advertise-routes on exit-nodes remotely.

    
On Skygate host (where Docker runs):

    
# Path to SSH key (must be reachable from the skygate container)
# Default: {{.SSHKeyPath}}
ssh-keygen -t ed25519 -f ~/.ssh/skygate_sync -N "" -C "skygate-sync"
ssh-copy-id -i ~/.ssh/skygate_sync.pub root@EXIT_NODE_IP

  


  
5. Register the node in Skygate

  

    In the form below specify the Headscale Node ID (visible on /admin/devices), Hostname, and SSH Target.
    The Accept routes field controls the --accept-routes flag on the node itself:
    for nodes that run other VPN services (Amnezia-AWG, OpenVPN, WireGuard, WG-Easy) choose false —
    otherwise Tailscale will pick up Google/Telegram routes from peers into the source-routing table 52 and traffic from the VPN container will go into a "black hole".
