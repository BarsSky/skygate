 Добавить Exit Node

  
  

    
1. Настройка Linux-хоста как exit-node

    
Выполнить на самом exit-узле (один раз):

    
# Включить IP forwarding
echo 'net.ipv4.ip_forward = 1' | sudo tee /etc/sysctl.d/99-tailscale-exit.conf
sudo sysctl -p /etc/sysctl.d/99-tailscale-exit.conf

# NAT для tailnet (100.64.0.0/10)
EXT_IF=$(ip route | awk '/default/ {print $5; exit}')
sudo iptables -t nat -A POSTROUTING -s 100.64.0.0/10 -o "$EXT_IF" -j MASQUERADE
sudo iptables -A FORWARD -i tailscale0 -o "$EXT_IF" -j ACCEPT
sudo iptables -A FORWARD -i "$EXT_IF" -o tailscale0 -m state --state RELATED,ESTABLISHED -j ACCEPT

# Сохранить правила
sudo mkdir -p /etc/iptables && sudo iptables-save | sudo tee /etc/iptables/rules.v4

  


  

    
2. Подключение Tailscale с exit-node флагами

    
Выполнить на exit-узле после регистрации в headscale:

    
tailscale up --login-server={{.ControlURL}} --ssh --advertise-exit-node --accept-routes

    

      --ssh — Tailscale SSH для удалённого управления

      --advertise-exit-node — режим exit-узла

      --accept-routes — принимать маршруты от других узлов
    

  


  

    
3. Windows-клиент: split-tunnel

    
На Windows-машине НЕ использовать --exit-node. Вместо этого:

    
tailscale up --accept-routes

    

      --accept-routes принимает advertised routes от exit-узлов — только whitelist-IP идут через туннель.

      Важно: НЕ использовать --exit-node на Windows — WireGuardNT (WFP) перехватывает весь трафик до таблицы маршрутизации, статические маршруты не работают.
    

  


  

    
4. SSH-доступ для автосинхронизации

    
Skygate использует SSH для удалённого обновления --advertise-routes на exit-узлах.

    
На Skygate-хосте (где запущен Docker):

    
# Путь к SSH-ключу (должен быть доступен в контейнере skygate)
# По умолчанию: {{.SSHKeyPath}}
ssh-keygen -t ed25519 -f ~/.ssh/skygate_sync -N "" -C "skygate-sync"
ssh-copy-id -i ~/.ssh/skygate_sync.pub root@EXIT_NODE_IP
