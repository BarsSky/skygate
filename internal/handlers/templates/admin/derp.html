Справка

Параметры:

  • Accepts — сколько раз derper принял WebSocket-апгрейд за всё время
  • Current conns — сколько WebSocket-соединений открыто прямо сейчас
  • Home conns — соединения от home-клиентов (чей home DERP = этот)
  • Clients total — уникальных клиентов, для которых этот DERP = home
  • Bytes/Packets — общий трафик (без разбивки по клиентам)

Почему клиент может не использовать этот DERP: Tailscale выбирает home DERP по минимальной UDP-latency. Если публичный DERP ближе, клиент его и выберет. Чтобы закрепить наш DERP на устройстве:

# Linux/macOS (Tailscale CLI):
sudo tailscale debug force-prefer-derp 900 (временно, до рестарта)
# Постоянно (через node-attrs):
sudo tailscale set --netfilter-mode=...
# Android/iOS:
Settings → Connection → Use DERP → Moscow (mow:900)
