Миграция на другой хост

После восстановления на новом сервере обязательно:
  1. Перейти в Настройки → Подключения и обновить URL/IP Headscale и публичный домен
  2. Проверить что HEADSCALE_URL в .env указывает на новый адрес headscale
  3. Если headscale API ключ изменился — обновить его в настройках
  4. Перезапустить skygate: sudo docker restart skygate
  5. Проверить что dashboard отображает корректные данные
Порядок миграции:
  1. На старом сервере: Сохранить бэкап (кнопка выше или ./scripts/backup.sh)
  2. Скопировать архив на новый сервер
  3. На новом сервере: установить Docker, клонировать git clone skygate
  4. Загрузить архив через эту страницу или выполнить ./scripts/migrate.sh archive.tar.gz
  5. Перейти в Настройки и обновить URL/IP
  6. Проверить Тесты — все шаги должны быть зелёными
