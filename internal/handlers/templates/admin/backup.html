Migration to another host

After restore on a new server, make sure to:
  1. Go to {{.T "settings.title"}} and update Headscale URL/IP and public domain
  2. Check that HEADSCALE_URL in .env points to the new headscale address
  3. If the headscale API key changed — update it in settings
  4. Restart skygate: sudo docker restart skygate
  5. Verify the dashboard shows correct data
Migration order:
  1. On the old server: create a backup (button above or ./scripts/backup.sh)
  2. Copy the archive to the new server
  3. On the new server: install Docker, clone the skygate repo
  4. Upload the archive via this page or run ./scripts/migrate.sh
  5. Go to Settings and update URL/IP
  6. Run the test suite — all steps should be green
